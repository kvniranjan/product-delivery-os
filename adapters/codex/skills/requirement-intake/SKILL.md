# Requirement Intake Skill

Use the matching workflow under `workflows/requirement-intake/`.

1. Read the workflow instructions.
2. Gather sanitized context.
3. Separate facts, assumptions, dependencies, risks, and open questions.
4. Produce the workflow output template.
5. Do not invent missing business rules or approvals.
