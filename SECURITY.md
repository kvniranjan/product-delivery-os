# Security and Privacy

This repository is public-facing by design. Treat it as unsafe for client, company, customer, production, or regulated data.

## Do Not Commit

- Client data
- Company data
- Customer data
- Account numbers, user IDs, emails, phone numbers, addresses, or personal data
- Credentials, API keys, tokens, certificates, or secrets
- Internal meeting transcripts, chat exports, or recordings
- Production data or production incident details
- Non-public architecture diagrams, internal URLs, environment names, hostnames, queue names, database names, or vendor implementation details
- Regulatory findings, audit evidence, legal advice, or confidential risk assessments

## Private Workspace

Use `workspace.example/` as a template only. Copy it to `workspace/` for real project work. `workspace/` is gitignored by default.

Before sharing any example publicly, replace sensitive details with fictional names and remove identifiers that could reveal a company, customer, vendor, system, or incident.

## AI Use

AI-generated artifacts must be reviewed by a human before delivery. Regulatory, legal, financial, medical, security, privacy, or compliance-sensitive outputs require expert review.

Do not paste confidential data into public AI tools. Use approved enterprise AI tooling and company policy where applicable.

## Reporting Issues

If you find a security or privacy issue, report it responsibly through repository maintainers. Do not publish sensitive details in a public issue.
